PHPstorm教程

日志

WHAT

PhpStorm是一个轻量级且便捷的PHP IDE，其旨在提供用户效率，可深刻理解用户的编码，提供智能代码补全，快速导航以及即时错误检查。

使用原因：

  • 功能强大！
  • 功能强大！
  • 功能强大！
    - 重要的事情要说三遍，刚刚入门，很多功能还在摸索中

安装

配置

一般就改一下编辑器风格和字体风格

  • Settings->Editor->Colors & Fonts->Font, 这里我选了Monokai，Sava As 以后才能改字体大小，习惯改到15
  • and so on

快捷键

  • Ctrl + Shift + Enter 添加完成标识符
  • Ctrl + P 显示参数信息
  • Ctrl + Q 快速查找函数文档
  • Ctrl + mouse 快速跳转到当前鼠标代码的源出处
  • Alt + Insert 快速添加魔术方法
  • Ctrl + O 选中并重写当前类的一个方法
  • Ctrl + I 实现一个魔术方法
  • Ctrl + Alt + T 需要回环的结构体
  • Ctrl + / 开启/关闭当前行的注释
  • Ctrl + Shift + / 开启/关闭当前块的注释
  • Ctrl + W 增量式的选中当前块
  • Ctrl + Shift + W 与ctrl+w相反
  • Ctrl + Alt + L 格式化选中代码块
  • Ctrl + Alt + I 自动缩进代码
  • Tab / Shift + Tab 添加/减少缩进
  • Ctrl + X or Shift + Delete Cut current line or selected block to clipboard
  • Ctrl + C or Ctrl + Insert Copy current line or selected block to clipboard
  • Ctrl + V or Shift + Insert Paste from clipboard
  • Ctrl + Shift + V 从最近的复制记录中粘贴…
  • Ctrl + D 复制粘贴当前行或块
  • Ctrl + Y 删除当前行
  • Ctrl + Shift + J Smart line join (HTML and JavaScript only)
  • Ctrl + Enter Smart line split (HTML and JavaScript only)
  • Shift + Enter 开启新行
  • Ctrl + Shift + U 选中块代码大/小写
  • Ctrl + Shift + ] / [ 增量选中块结构标识符代码段
  • Ctrl + Delete 删除一个单词至结尾
  • Ctrl + Backspace 删除一个单词至词头
  • Ctrl + NumPad+/- 折叠代码段
  • Ctrl + Shift + NumPad+ 折叠所有
  • Ctrl + Shift + NumPad- 开启所有折叠
  • Ctrl + F4 关闭当前tab
  • Ctrl + N 寻找一个类
  • Ctrl + Shift + N 寻找一个文件
  • Ctrl + Alt + Shift + N 寻找一个符号
  • Alt + Right/Left 去上一个/下一个tab框
  • F12 返回前一个工具栏
  • Esc 返回编辑框
  • Shift + Esc 隐藏其他活动窗口
  • Ctrl + G 去到指定行
  • Ctrl + E 最近编辑的文件
  • Alt + F1 当前文件的一些选择模式，很有用
  • Ctrl + B or Ctrl + Click 代码源出处
  • Alt + Up/Down 去上一个/下一个方法体
  • Alt + Home 显示文件层级浏览工具
  • F11 开启/关闭书签
  • Ctrl + F11 开启/关闭数字书签
  • Ctrl + #[0-9] 中转到数字书签
  • Shift + F11 显示所有的书签

Tools-vagrant

  1. PHPStorm自带vagrant插件，可在setting->Plugns中开启禁用
  2. 通过 Tools->Vagrant 即可方便操作vagrant 基本操作
  3. 由于我的 vagrant 目录非工程目录，通常通过Tools->Start SSH sessions连接vagrant虚拟机
  4. 127.0.0.1' '2222' 'vagrant' 'vagrant'，可以设置在Setting->Build,Execution,Deployment->Deployment`,保存为SFTP协议，
    advanced中编码设为UTF-8
  5. 即可方便的在编辑器中写脚本，Terminal中跑脚本

安装过程碰到的问题

安装打开工程路径，两个worning，两个error：
Can't use Subversion command line client: svn
TortoiseSVN默认不支持命令行command line

解决办法：

  • 下载一个命令行工具 sliksvn
    https://sliksvn.com/download
  • 一路next，安装完成
  • 在use conmand line client设置成你的C:\Program Files\SlikSvn\bin\svn.exe即可使用svn了

另外一个error
Errors found while svn working copies detection. Fix it.
svn版本不同

svn问题解决不了，以上的方法都没有解决，应该是windows下的命令设置正确

其他问题

总结

有人说IDE过大，vim以及sublime比较轻便，扩展性更好，更适合程序员，我也去专门学习了一下vim以及sublime快捷键
这么说吧，首先，习惯是一件很可怕的事情，一件工具的好坏不是以某些人习不习惯来决定的，很多东西也是不分好坏的，选择适合自己的并去习惯就好了，、然而很多人因为习惯去放弃尝试，这就是一件可怕的事情，世界是在发展的，你可以选择不改变，但请你不要阻止别人改变我喜欢在PHPstorm上打个勾，而不喜欢像vim或者sublime那样为了某个配置去上网查资料，改变一个个的文件。不便捷带来的其他人看不懂并不能成为你装X的资本再来谈谈IDE过大的问题，我最早接触手机的时候，诺基亚系统，一个应用只有1M不到，到后来的Android系统，也是很小的一个包，到现在的应用都十几M，几十M软件是在发展的，硬件也是在发展的，我在我的ssd 16G内存台式机上，也是可以秒开PHPstorm，并且完全不卡，PHPstorm还可以装各种插件，基本会集成PHP开发会接触到的所有新技术，写了也没准备跟别人争论什么，只是在陈诉一些自以为是的“事实”，大家开心就好> 人活着嘛，最重要的就是开心啦 —
