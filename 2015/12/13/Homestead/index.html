日志

5.1Homestead
需要：
vagrant1.7.X
VirtualBox5.X

  • shadowsocks+proxifier,全局走socks5代理
  • vagrant init laravel/homestead-7; vagrant up --provider virtualbox
  • or vagrant box add laravel/homestead-7,选择1
  • or Downloading: https://atlas.hashicorp.com/laravel/boxes/homestead-7/versions/0.2.0/providers/virtualbox.box

以下为直接升级到PHP7的做法

vagrant box add laravel/homestead-7

  • 不翻墙一般是下不来的，可以去直接下载然后add进去
    https://atlas.hashicorp.com/laravel/boxes/homestead-7/versions/0.2.0/providers/virtualbox.box
    ps:用国外服务器下载真的好快
    • vagrant box add laravel/homestead-7 virtualbox-homestead7.box

git clone -b php-7 https://github.com/laravel/homestead.git Homestead7
bash init.sh
会生成一个~\.homestead文件夹下生成一个 Homestead.yaml文件

  • 最近经常见到yaml格式的文件

这个文件将vagrant的配置简单化了，个人觉得
还比较容易能够理解
一般改一下共享文件夹，以及sites，就可以了

这之前还需要生成一个ssh密钥，由于安装了git并且覆盖了windows的命令行，直接在命令行中
ssh-keygen -t rsa -C "1096364408@qq.com"

指明ssh密钥路径，默认

共享文件夹，其实可以把你的所有代码的总目录共享进去，每个site对应到不同的子目录，
Vagrant 默认的 VirtualBox 共享目录方式读写性能表现并不好，好在 Vagrant 支持 NFS 文件系统方式的共享，我们可以启用 NFS 提升性能。
只需在 folders中加上type: "nfs",
你可以通过配置 hhvm 属性为 true 来让虚拟站点支持 HHVM:Z,在sites下面增加hhvm: true

配置hosts文件，如：192.168.10.10 homestead.app

vagrant up
vagrant destroy –force

还可以将homestead对单独的项目创建单独的虚拟机
composer require laravel/homestead –dev
会在项目目录下生成Vagrantfile和Homestead.yaml文件，make命令会自动去配置

Vagant ssh

可以配置别名alias vm="ssh vagrant@127.0.0.1 -p 2222"

连接数据库
默认安装了Mysql和Postgres两种数据库，可以在本机通过端口连接，33060(Mysql),54320(Postgres),账号密码是homestead / secret

在Homestead环境上架运行后，可能会增加Nginx站点，可以通过两种方法
1.在Homestead.yaml文件中增加站点然后执行vagrant privision

也可以自定义转发额外的端口到Vagrant box，

Blackfire Profiler

vagrant up
vagrant 启动连接超时，报错如下

去virtualBox查看，报错如下
VT-x/AMD-V 硬件加速在您的系统中不可用。您的 64-位虚拟机将无法检测到 64-位处理器，从而无法启动。

网上找的解决方法：

进入你的BIOS，然后将virtualization（虚拟化）设置为enable即可
报错咯：

网上找的解决方法，发现没用

vagrant ssh登陆后发现共享文件夹没了，无奈Google之，之后找到两种方法。
1,sudo /etc/init.d/vboxadd setup
2,sudo apt-get install virtualbox-guest-utils
vagrant reload后问题完美解决。

stackoverflow上解决方法：
vagrant ssh
sudo ln -s /opt/VBoxGuestAdditions-4.3.10/lib/VBoxGuestAdditions /usr/lib/VBoxGuestAdditions
vagrant reload
问题依旧

http://docs.vagrantup.com/v2/synced-folders/nfs.html

Windows users: NFS folders do not work on Windows hosts. Vagrant will ignore your request for NFS synced folders on Windows.

看到这样一句，热泪盈眶
第一次启动的时候最好翻墙，

以下为4.2的homestead安装方法
composer global require "laravel/homestead=~2.0"

php5.3、5.4和apache都是用vc9编译，电脑必须安装vc9运行库才能运行。
php5.5、5.6是vc11编译，如用php5.5、5.6必须安装vc11(32位)运行库。
http://www.phpstudy.net/a.php/184.html

重新安装composer一遍，安装的时候选择了php位置

安装过程报错：

下载安装ca证书，或者php版本从5.6改为5.5
ca证书：

  • http://curl.haxx.se/docs/caextract.html
  • openssl.cafile=C:/lamp/php56/verify/cacert.pem

安装好Homestead命令行，需要将~\AppData\Roaming\Composer\vendor\bin加入环境变量，win10下

homestead init
