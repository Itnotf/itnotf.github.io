Vagrant教程

日志

WHAT

Vagrant是一个基于Ruby的工具，用于创建和部署虚拟化开发环境。它 使用Oracle的开源VirtualBox虚拟化系统，使用 Chef创建自动化虚拟环境。

使用Vagrant的原因：

  • 熟悉linux环境，方便、不占资源的使用虚拟机
    - 了解一下新技术

安装

  • 环境：win10 专业版
  • 版本：Oracle VM VirtualBox 4.3.24 Vagrant 1.7.2
  • box : precise64

安装完成后，使用以下命令

  • vagrant init
    目录下会出现Vagrantfile文件，里面可以修改各种配置
  • vagrant add precise64 precise64.box
    添加box，每个box就是一个linux系统，我安装的是Ubuntu12.04
  • vagrant box list
  • vagrant status
  • vagrant up

问题

  1. 一般情况下vagrant up都会出现问题，可能是版本问题，系统问题等等问题
    比如说最开始安装了4.3.30版本，启动失败，报错：
    default: Warning: Connection refused.retrying……
    default: Warning: Connection refused.retrying……
    default: Warning: Connection refused.retrying……
    default: Warning: Connection refused.retrying……
    default: Warning: Connection refused.retrying……
    一开始以为是ssh的问题，Windows没有ssh命令，需要安装ssh客户端，git中带了ssh客户端，安装git，并在环境变量中添加了bin的路径。错误没有解决。继续Google
    更换VirtualBox为4.2.16版本。问题解决。

  2. ssh问题，登陆不上虚拟机，windows 上不知道自己的ssh装在哪里了……
    我的解决方法：修改Vagrantfile，直接用账号密码登陆

  3. 端口映射，将虚拟机的端口映射本地从而实现类似http://localhost:80这种访问方式，这种方式比较麻烦，Host-only比较
    修改Vagrantfile的一下一行注释去掉，并保存
    config.vm.network :private_network, ip: "192.168.33.10"
    即可通过 192.168.33.10来访问这台虚拟机，
    但是， 我在win7上这么做没问题，升级到win10后，无法启动虚拟机了
    解决方法：暂时使用端口映射……

  4. 现在启动Vagrant的时候都会提醒我的Vagrant和Vm版本不对应，需要4.2的版本，暂时没升级Vagrant，或者降级Vm
  5. 使用precise32box的时候，第一次vagrant up会不断提示超时，后来找到原因，这个box的Ubuntu系统第一次进入系统时，会有个系统选择界面，没有默认进入，会一直卡住
    解决方法：直接用VM启动改虚拟机，选择第一个系统进入，然后输入账号密码都是vagrant，正常关机sudo poweroff，以后就可以使用vagrant up了

以后继续研究完善

使用

一般情况下，使用 vagrant up vagrant halt 来开启关闭虚拟机，然后可以使用ssh工具登录使用linux系统，共享本地文件夹
其他命令

  • vagrant package 打包命令
  • vagrant reload
  • vagrant ssh
  • vagrant destroy
  • vagrant box list
  • vagrant box remove XXX
  • vagrant box add –name box_name /path/of/box/file

总结

之前不太使用虚拟机的，电脑太差，一开虚拟机就卡的要死，vagrant确实方便了使用虚拟机以及各种配置，
最重要的还是方便统一部署，说起来这个算是运维的工作吧，我是搞php的，就暂且认为学习Linux是有必要的吧，自己安装各种软件服务，配置都稍微熟悉一下
vagrant的英文意思是流浪者，以后你可以带着你自己打包的一个box去不同的机器上面，却能够有着自己习惯的工作环境，这就是其意义所在
这篇算起来仅仅是安装教程，具体的使用其实也蛮简单，需要的时候再查一下吧，28定律，我这里写的是2
