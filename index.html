php-fpm权限问题

php-fpm是运行在nobody用户下的，
nginx是运行在www-data用户下的，
web目录用户是web，

Web服务器运行用户和Web目录所有者不同是最基本的安全原则，要不就会像在Windows开发那样，PHP脚本可以对Web目录甚至Windows系统目录内容进行修改，完全没有安全性可言。

2015-12-05

php7安装

下载

去官网上找一下镜像地址

编译安装

  • 根据自己的需要设定配置项
    可以使用命令./configure --help查看可配置项
    以下为我的配置（网上找的，基本上是一些常用扩展，懒得一个个看了，先全部装了再说）
    突然想到用某哥们儿的配置，没有加前缀，然后一堆文件夹散落各个地方……

  • 配置好后执行make，&&不要写错了
    make && make install
    安装过程中经常会提示一些库不存在，安装好库后继续安装重新编译
    开始的时候我看到网上教程经常有一句命令是安装一堆库，有的教程又没有，现在总算知道原因了
    安装到某一次的时候提示内存不足，杀了一些进程之后重新编译。

  • 以上，php7已经编译安装完成啦
    还需要做一些配置，如下：

  • 开启Opcache模块
    vim /usr/local/php7/lib/php.ini/

  • 安装Zend Opcache
    yum install php-pecl-zendopcache
    配置文件

vi /etc/php.d/opcache.ini

  • 启动php7-fpm
    /etc/init.d/php7-fpm start

  • 添加环境变量
    vim /etc/profile

  • 测试php版本
    php -v

Others
