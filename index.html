Xmanager Enterprise 5

之前一直用的是X系列的Xshell、Xftp，最近发现了一个集成的软件包。一次注册就可以了，当然我是没钱买的，2333。
安装完后会有好几个软件。

###WHAT

####Xmanager

Xmanager是一个简单易用的高性能的运行在Windows平台上的X Server软件。它能把远端Unix/Linux的桌面无缝地带到你的Windows上，甚至当你的电脑是在内网或防火墙后，你也能通过SSH协议安全的运行远端的X应用程序。

####Xbrowser

Xbrowser 使用 xdcmp 协议通过图形化桌面远程连接到Unix/Linux主机。

####Xftp

Xftp是一个用于MS Windows平台的强大的FTP和SFTP文件传输程序。Xftp让你能安全地在Unix/Linux和Windows PC之间传输文件。

####Xlpd

Xlpd是一个用于MS Windows平台的LPD(行式打印机虚拟后台程序)应用程序。安装了Xlpd后，你的带有打印机的本地PC就成为了一个打印服务器，来自不同远程系统的打印任务都能在网络环境中得到请求和处理。

####Xshell

Xshell是一个用于MS Windows平台的强大的SSH，TELNET，和RLOGIN终端仿真软件。它使得用户能轻松和安全地从Windows PC上访问Unix/Linux主机。

####Xstart

Xstart通过图形化界面连接到Unix/Linux主机。

###安装
去官网下个最新的Xmanager Enterprise 5 安装即可。
打开的时候会让输入注册码，下个注册机填一下。

###总结之前一直只在用Xshell以及Xftp，以后将其他的软件也都试试。

Homestead

WHAT

就我理解，就是Laravel自己做的一个vagrant box,里面有各种安装好的软件和配好的环境。
不仅仅作为Laravel的环境，平时做PHP开发，也可以使用Homestead这个box，能简单的共享文件夹，配置站点。
当然我说的是专注于开发可以简单的这么做，如果想了解每一个软件的编译安装，熟悉各种环境配置，就找一个纯净的box也是可以的。

安装

  • 环境
    • win10
    • vagrant 1.7.X
    • VirtualBox5.X
  • 版本
    • Homestead-7

vagrant和Virtualbox都安装好了，如果网络允许的话直接add laravel/homestead-7这个box就好了.
刚刚php7出来了，所以直接安装了新版本，也有之前的版本laravel/homestead php5.6

  • shadowsocks+proxifier,全局走socks5代理
  • vagrant init laravel/homestead-7; vagrant up --provider virtualbox
  • or vagrant box add laravel/homestead-7,选择1
  • or Downloading: https://atlas.hashicorp.com/laravel/boxes/homestead-7/versions/0.2.0/providers/virtualbox.box
  • vagrant box add laravel/homestead-7 virtualbox-homestead7.box

可以直接走代理安装，box大概快1G了，也可以浏览器代理下载，然后使用vagrant 命令加进去。
ps:用国外服务器下载真的好快

以上就是homestead的box安装了，homestead在github上还有一个项目。

  • git clone -b php-7 https://github.com/laravel/homestead.git Homestead7
  • bash init.sh

安了git后，执行以上两步就clone下来homestead的项目，
初始化后会在~\.homestead文件夹下生成一个 Homestead.yaml文件
最近经常见到yaml格式的文件，感觉比json、xml都直观一些呀

Homestead.yaml

这个文件将vagrant的配置简单化了，个人觉得
还比较容易能够理解
一般改一下共享文件夹，以及sites，就可以了

这之前还需要生成一个ssh密钥，由于安装了git并且覆盖了windows的命令行，直接在命令行中
ssh-keygen -t rsa -C "1096364408@qq.com"

指明ssh密钥路径，默认

共享文件夹，其实可以把你的所有代码的总目录共享进去，每个site对应到不同的子目录，
Vagrant 默认的 VirtualBox 共享目录方式读写性能表现并不好，好在 Vagrant 支持 NFS 文件系统方式的共享，我们可以启用 NFS 提升性能。
只需在 folders中加上type: "nfs",
你可以通过配置 hhvm 属性为 true 来让虚拟站点支持 HHVM:Z,在sites下面增加hhvm: true

配置hosts文件，如：192.168.10.10 homestead.app

启动和关闭

  • vagrant up
  • vagrant destroy --force

还可以将homestead对单独的项目创建单独的虚拟机
composer require laravel/homestead –dev
会在项目目录下生成Vagrantfile和Homestead.yaml文件，make命令会自动去配置

  • Vagant ssh

可以配置别名alias vm="ssh vagrant@127.0.0.1 -p 2222"

连接数据库
默认安装了Mysql和Postgres两种数据库，可以在本机通过端口连接，33060(Mysql),54320(Postgres),账号密码是homestead / secret

在Homestead环境上架运行后，可能会增加Nginx站点，可以通过两种方法

  1. 在Homestead.yaml文件中增加站点然后执行vagrant privision
  2. 也可以自定义转发额外的端口到Vagrant box，

Blackfire Profiler

php-fpm权限问题

php-fpm是运行在nobody用户下的，
nginx是运行在www-data用户下的，
web目录用户是web，

Web服务器运行用户和Web目录所有者不同是最基本的安全原则，要不就会像在Windows开发那样，PHP脚本可以对Web目录甚至Windows系统目录内容进行修改，完全没有安全性可言。
PHP-FPM运行用户可以也建议跟Nginx用户相同

所以拿到post过来的数据直接写入web目录是没有权限的。

  • 给目录赋予nobody的写权限
  • 写入其他目录中，之后用shell脚本处理

  1. PHPStorm自带vagrant插件，可在setting->Plugns中开启禁用
  2. 通过 Tools->Vagrant 即可方便操作vagrant 基本操作
  3. 由于我的 vagrant 目录非工程目录，通常通过Tools->Start SSH sessions连接vagrant虚拟机
  4. 127.0.0.1' '2222' 'vagrant' 'vagrant'，可以设置在Setting->Build,Execution,Deployment->Deployment`,保存为SFTP协议，
    advanced中编码设为UTF-8
  5. 即可方便的在编辑器中写脚本，Terminal中跑脚本
